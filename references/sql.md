# sql

`sql` 是只读 SQL 查询入口，用于查询工作空间索引数据库。

## 子命令一览

- 直接调用：`sql <statement>`

## 子命令详述

| 子命令 | 用途 | 语法 | 关键参数 | 输出/副作用 | 备注 |
|---|---|---|---|---|---|
| `sql <statement>` | 执行单条只读 SQL 查询 | `siyuan sql <statement> -w <workspace>` | `<statement>`：必填，只取第一个位置参数作为 SQL 语句，包含空格时需按当前 shell 规则加引号；`--limit`, `-l`：可选，最大返回行数，默认 100，小于 1 时按 100 处理，SQL 自带 `LIMIT` 时优先生效 | 只读；默认按列名排序后输出表格和行数，空结果输出 `No results.`，`-f json` 时返回缩进后的行对象数组 | 无子命令；只允许单条 `SELECT` 或 `WITH` 查询，非查询只读语句、写语句或多语句会被拒绝；`--dry-run` 对该命令无预览差异 |

## 通用注意事项

- 该命令读取 CLI 初始化后的索引数据库，不用于直接修改工作空间数据。
- 末尾分号后只有空白或 SQL 注释时仍按单条语句处理；分号后还有可执行语句时会按多语句拒绝。
